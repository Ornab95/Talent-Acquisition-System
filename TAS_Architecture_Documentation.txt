===============================================================================
                    TALENT ACQUISITION SYSTEM (TAS) ARCHITECTURE
===============================================================================

OVERVIEW
--------
A comprehensive, enterprise-grade talent acquisition platform built with modern
web technologies. Features complete recruitment workflow from job posting to
hiring, with advanced analytics, interview management, and automated notifications.

===============================================================================
TECHNOLOGY STACK
===============================================================================

BACKEND
-------
- Framework: Spring Boot 3.4.10
- Language: Java 21
- Database: MySQL 8.0+ with JPA/Hibernate
- Security: Spring Security with JWT (JJWT 0.12.3)
- Email: Spring Boot Mail with Thymeleaf templates
- Build Tool: Maven 3.6+
- API Documentation: SpringDoc OpenAPI 2.8.5
- Validation: Spring Boot Validation
- File Upload: Multipart support (10MB limit)

FRONTEND
--------
- Framework: Angular 20.3.0
- Language: TypeScript 5.9.2
- Styling: Tailwind CSS 4.1.14
- UI Components: PrimeNG 20.2.0, Angular Material 20.2.8
- Icons: PrimeIcons 7.0.0
- HTTP Client: Angular HttpClient with Interceptors
- Routing: Angular Router with Guards
- State Management: RxJS 7.8.0 Observables
- Build Tool: Angular CLI 20.3.5

===============================================================================
DATABASE SCHEMA
===============================================================================

CORE ENTITIES
-------------
1. users
   - id, email, password_hash, first_name, last_name
   - role (CANDIDATE, HR_ADMIN, RECRUITER, HIRING_MANAGER, SYSTEM_ADMIN)
   - department, phone, created_at, updated_at

2. jobs
   - id, title, description, requirements, location, department
   - min_salary, max_salary, deadline, active
   - posted_by_id (FK to users), created_at, updated_at

3. applications
   - id, job_id (FK), candidate_id (FK to users)
   - cover_letter, resume_file_path, portfolio_url
   - status (APPLIED, REVIEWED, SHORTLISTED, REJECTED)
   - screening_score, source_channel, assigned_recruiter
   - applied_at, last_updated, notes

4. interviews
   - id, application_id (FK), interviewer_id (FK to users)
   - scheduled_at, interview_type (PHONE, VIDEO, ONSITE)
   - status (SCHEDULED, COMPLETED, CANCELED)
   - feedback, meeting_link, created_at

5. offers
   - id, application_id (FK), salary, position_title
   - start_date, offer_expiry_date
   - status (PENDING, ACCEPTED, REJECTED, EXPIRED)
   - benefits, terms, created_at, responded_at

6. companies (for recruiter registration)
   - id, company_name, company_email, description
   - website, industry, location, created_at

API ENDPOINTS
===============================================================================

AUTHENTICATION
--------------
POST /api/auth/login          # User login
POST /api/auth/register       # User registration
POST /api/company/register    # Company registration

JOB MANAGEMENT
--------------
GET    /api/jobs/public       # Get all active jobs
GET    /api/jobs/{id}         # Get job by ID
POST   /api/jobs              # Create new job (authenticated)
PUT    /api/jobs/{id}         # Update job (authenticated)
DELETE /api/jobs/{id}         # Delete job (authenticated)

APPLICATION MANAGEMENT
----------------------
POST /api/applications/apply  # Submit job application
GET  /api/applications/job/{jobId}  # Get applications by job
PUT  /api/applications/{id}/status  # Update application status

INTERVIEW MANAGEMENT
--------------------
POST /api/interviews/schedule           # Schedule interview
GET  /api/interviews/application/{id}   # Get interviews by application
PUT  /api/interviews/{id}/feedback      # Add interview feedback
PUT  /api/interviews/{id}/status        # Update interview status
GET  /api/interviews/upcoming/{id}      # Get upcoming interviews

OFFER MANAGEMENT
----------------
POST /api/offers/create                 # Create job offer
GET  /api/offers/application/{id}       # Get offers by application
PUT  /api/offers/{id}/status            # Update offer status
GET  /api/offers/all                    # Get all offers
DELETE /api/offers/{id}                 # Delete offer

ADMIN FUNCTIONS
---------------
GET  /api/admin/users         # Get all users
POST /api/admin/users         # Create user
PUT  /api/admin/users/{id}    # Update user
GET  /api/admin/analytics     # Get system analytics

SECURITY IMPLEMENTATION
===============================================================================

AUTHENTICATION
--------------
- JWT-based authentication with Bearer tokens
- Password hashing using BCrypt
- Token expiration: 24 hours (86400000ms)
- Secure token storage in localStorage

AUTHORIZATION
-------------
- Role-based access control (RBAC)
- Route guards: AuthGuard, RoleGuard, GuestGuard
- Method-level security with @PreAuthorize annotations
- CORS configuration for cross-origin requests

INPUT VALIDATION
----------------
- Backend: Bean Validation with @Valid annotations
- Frontend: Template-driven validation with ngModel
- SQL injection prevention with parameterized queries
- XSS protection with proper data sanitization
